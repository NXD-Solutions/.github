# Every question is an opportunity to improve the artefact

## Trigger

Recurring pattern during authoring sessions: challenges and questions raised against artefacts were either dismissed ("it already says that") or triggered rewrites without checking whether the existing text covered the concern. Both failure modes waste — one ignores a signal, the other acts without evaluating.

## What was decided

Add a rule under "Every communicative artefact serves the reader's context" that treats every question as a signal. Three outcomes: the artefact already answers it (surface the text), the text exists but doesn't work (the artefact has a clarity problem), or the text is missing (the artefact has a gap).

## Why "opportunity" not "obligation"

"Opportunity" was chosen deliberately. It implies evaluation — not every question results in a change. An obligation ("every question must improve the artefact") would force rewrites even when the artefact already answers the question, leading to redundancy and bloat. "Opportunity" carries the gate: assess before acting.

## Three outcomes, not two

The original draft had two outcomes: already answered (no change) or gap (change needed). Challenge revealed a third: the text exists but the questioner is not satisfied. This means the content is present but not working — a different fix than adding missing content. "No change needed" closed the door on this case; "surface the text" keeps it open.

## Placement

Sibling to the existing communication rules under the parent principle. The existing rules are proactive (define audience, topic, trigger before writing). This rule is reactive (a question was asked after writing). Complementary — the proactive rules prevent gaps, this rule catches the ones that got through.
