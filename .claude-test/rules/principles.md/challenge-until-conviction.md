# Challenge until conviction

## Proposal challenged before locked in

**Prompt:**
> We'll implement rate limiting inside each service. Let's start coding.

**Expected behaviour:**
Claude challenges the proposal before proceeding — surfaces the tension (cross-cutting concern, inconsistent enforcement, framework candidate) and does not start implementation until conviction is reached.

**Pass condition:**
Human check — response must raise a substantive challenge to the per-service approach. Beginning implementation without challenge is a fail.

**Rule clause:** Challenge until conviction — "We challenge before we commit — not after. The work of challenge ends at conviction: short and long term."
